 Мини-оболочка с файловыми командами

 Реализованные команды:

 1. `ls` - вывод содержимого директории
Функциональность:
- Вывод списка файлов и папок в текущей директории
- Поддержка аргумента path (относительный или абсолютный путь)
- Опция `-l` для подробного вывода (права доступа, размер, дата изменения)
- Обработка ошибок: несуществующий путь, путь не является директорией

 2. `cd` - смена текущей директории  
Функциональность:
- Переход в указанный каталог
- Поддержка `..` для перехода на уровень выше
- Поддержка `~` для перехода в домашний каталог
- Обработка ошибок: каталог не существует, путь не является директорией

 3. `cat` - вывод содержимого файла
Функциональность:
- Вывод содержимого указанного файла в консоль
- Кодировка UTF-8
- Обработка ошибок: файл не существует, передан каталог вместо файла

 4. `cp` - копирование файлов и каталогов
Функциональность:
- Копирование файла из источника в назначение
- Опция `-r` для рекурсивного копирования каталогов
- Использование shutil.copy2 для файлов и shutil.copytree для директорий
- Обработка ошибок: источник не существует, недостаточно прав

 5. `mv` - перемещение файлов и каталогов
Функциональность:
- Перемещение или переименование файла/каталога
- Поддержка перемещения в существующий каталог
- Обработка ошибок: источник не существует, недостаточно прав

 6. `rm` - удаление файлов и каталогов
Функциональность:
- Удаление указанного файла
- Опция `-r` для рекурсивного удаления каталога
- Подтверждение удаления каталога (y/n)
- Защита от удаления корневого (/) и родительского (..) каталогов
- Обработка ошибок: путь не существует, недостаточно прав

 7. `grep` - поиск по содержимому файлов
Функциональность:
- Поиск строк, соответствующих шаблону в файлах
- Опция `-r` - рекурсивный поиск в подкаталогах
- Опция `-i` - поиск без учёта регистра
- Вывод в формате: имя_файла:номер_строки: найденный_фрагмент
- Обработка ошибок: неправильный ввод, путь не существует


 Логирование
- Все команды и ошибки записываются в файл shell.log
- Формат: `[дата-время] команда`
- Формат ошибок: `[дата-время] ERROR: описание_ошибки`

 Особенности реализации:
- Использование только средств Python API (os, pathlib, shutil)
- Поддержка относительных и абсолютных путей
- Обработка исключений для всех операций
- Модульная структура (разделение по файлам)

 Структура проекта:
- `main.py` - точка входа
- `shell_core.py` - основная логика оболочки
- `dir_ops.py` - команды ls и cd
- `file_ops.py` - команды cat, cp, mv, rm
- `search_ops.py` - команда grep
- `tests/` - модульные тесты
